What is Claude Code?

Not a copilot.
An agent.

  • CLI tool that reads your codebase, plans, and acts
  • Writes code, runs commands, iterates — without step-by-step instruction
  • Reads CLAUDE.md at startup to understand your project
  • Uses tools: file read/write, bash, web search, sub-agents
  • Works on the task until it's done — not just the next line

Sub-Agents

One prompt.
Multiple specialists.

Orchestrator

Main Agent

Reads CLAUDE.md, plans the pipeline, spawns and coordinates children

Sub-Agent A

Specialist

Focused task, isolated context window, writes output to a file

Sub-Agent B

Specialist

Runs in parallel with A, separate context — no interference

Sub-Agent C

Synthesizer

Runs after A and B, reads their output files, produces final report

Each sub-agent gets a focused prompt and its own context — so the analysis stays sharp and doesn't blur across tasks.
Why Multi-Agent?

Three reasons it
outperforms one big prompt

  • Parallelism — agents run simultaneously, wall time drops
    Team profiling, matchup analysis, and upset detection all happen at once
  • Specialization — each agent has one job, done deeply
    A matchup analyst isn't distracted by narrative concerns
  • Composability — the pattern transfers to any problem
    Point the same architecture at sales data, clinical trials, financial reports
The Pattern

## Agent Architecture

### Sub-Agent 1 — Data Gatherer
**Inputs:** sec_teams.csv, bracket.json
**Outputs:** Write to output/data_audit.md

### Sub-Agent 2 — Team Profiler (parallel)
**Outputs:** Write to output/team_profiles.md

### Sub-Agent 5 — Narrative (runs last)
**Inputs:** Read output/team_profiles.md ...
**Outputs:** Write output/tournament_preview.md
  • Markdown file — no Python orchestration code
  • Define roles, data paths, output expectations
  • Portable — swap the dataset, keep the architecture
